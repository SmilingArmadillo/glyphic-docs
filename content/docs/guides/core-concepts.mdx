# Core Concepts

## The render pipeline

Glyphic processes your diagram source in three stages:

```
Source text
  → Meta-block parser   (@animate, @style, @compare, etc.)
  → Mermaid renderer    (produces SVG)
  → Injector pipeline   (applies animations, styles, overlays)
  → Preview panel       (displays result)
```

## Meta-blocks

Meta-blocks are Glyphic-specific extensions appended after valid Mermaid source. They use an `@directive ... @end` syntax:

```
@animate
  A -> B: duration=1s
@end
```

Mermaid ignores them. Glyphic's parser strips them before passing source to Mermaid, then applies the instructions to the rendered SVG.

## Supported directives

| Directive | Purpose |
|---|---|
| `@animate` | Edge and node animations |
| `@style` | Node fill, stroke, text color |
| `@status` | Live status overlay |
| `@compare` | Before/after split view |
| `@present` | Presentation overlays |
| `@simulate` | Traffic simulation |

## Plugin system

Renderers are swappable via the `RendererPlugin` interface. The default renderer is Mermaid. A Prettify renderer (R DiagrammeR) can be activated from the toolbar.
