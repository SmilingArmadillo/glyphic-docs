The `@animate` block controls animations on individual edges and nodes. Each line targets either an edge (`A-->B`) or a node (`NodeId`).

```text
@animate
Client-->Server: type=line-flow, color=#38bdf8, speed=2
Server: type=node-pulse, pulseStyle=ripple, color=#ef4444, speed=1.4
```

## Edge animations

```text
@animate
{from}-->{to}: type={type}, [param=value, ...]
```

<Accordions>
  <Accordion title="type=line-flow — segment draws on, holds, then retracts">

A segment of the edge animates: draws on, holds, then retracts and fades. Repeats indefinitely.

```text
@animate
Client-->Server: type=line-flow, color=#38bdf8, speed=2, preset=balanced, glow=subtle
```

**Presets:**

| Preset | Character |
|--------|-----------|
| `flash` | Short segment, fast travel — urgent, rapid traffic |
| `balanced` | Medium segment, moderate speed — default working state |
| `long-hold` | Long segment, slow travel — steady, high-volume throughput |

  <Accordion title="type=pill — rounded rectangle travels tangent-aligned">

A rounded rectangle travels along the edge, rotating to stay tangent-aligned to the path direction. Supports an optional halo glow.

```text
@animate
A-->B: type=pill, color=#4fc3f7, speed=1.5
B-->C: type=pill, color=#7ee787, speed=1, glow=strong
C-->D: type=pill, color=#f59e0b, speed=0.8, glow=subtle, width=24, height=12
```

## Node animations

Node animations target a node by its ID (not an edge).

<Accordions>
  <Accordion title="type=node-pulse — pulsing border, fill, shadow, flash, or ripple">

```text
@animate
{nodeId}: type=node-pulse, [param=value, ...]
```

```text
@animate
Server: type=node-pulse, pulseStyle=border, color=#ef4444, speed=1.5
Server: type=node-pulse, pulseStyle=flash,  color=#ef4444, speed=1
Server: type=node-pulse, pulseStyle=ripple, color=#ef4444, speed=1.4, rippleCount=3
```

<TypeTable
  type={{
    pulseStyle: {
      description: "Visual effect",
      type: '"border" | "fill" | "shadow" | "flash" | "ripple"',
      default: "border",
    },
    color: {
      description: "Pulse or ring color",
      type: "CSS color",
      default: "#3b82f6",
    },
    speed: {
      description: "Duration of one cycle in seconds",
      type: "number",
      default: "1.5",
    },
    rippleCount: {
      description: "Number of staggered rings (ripple style only)",
      type: "1 | 2 | 3",
      default: "1",
    },
  }}
/>

**`pulseStyle` variants:**

| Style | Effect | Use case |
|-------|--------|----------|
| `border` | Smooth opacity oscillation on a border ring | Persistent warning, attention state |
| `fill` | Animates the fill opacity of the node shape | Highlighting, selection |
| `shadow` | Border ring with Gaussian blur glow | Emphasis, "active" state |
| `flash` | Hard on/off strobe | Urgent alert, error state |
| `ripple` | Expanding rings radiate outward and fade | Broadcasting, active transmission |

<Callout type="info">
**`flash` color tip:** Choose a color that contrasts with the node's static border. If the flash color matches the existing `classDef` stroke or `@style outline` color, the strobe will be invisible.
</Callout>

  </Accordion>
  <Accordion title="type=oval-inset — filled oval in the left cap of a stadium node">

A filled oval injected into the left cap of a stadium (pill-shaped) node. Use with nodes declared using the `([text])` stadium syntax.

```text
@animate
S1: type=oval-inset, color=#22c55e
```

<TypeTable
  type={{
    color: {
      description: "Fill color of the oval",
      type: "CSS color",
      default: "#3b82f6",
    },
  }}
/>

<Callout type="warn">
The node must use stadium shape (`([NodeId])`) syntax in the flowchart. On standard rect-shaped nodes the oval will appear misaligned.
</Callout>

Combine with `classDef` for full control over border and fill:

```text
flowchart LR
  S1([S1])
  classDef serverNode fill:#052e16,stroke:#22c55e,stroke-width:2px,color:#e2e8f0
  class S1 serverNode

@animate
S1: type=oval-inset, color=#22c55e
```

  </Accordion>
  <Accordion title="type=status-dot — animated indicator dot on any node">

A small animated dot injected into or adjacent to any node shape. Supports four animation styles and seven positions.

```text
@animate
GW: type=status-dot, style=breathe, color=#22c55e, position=left, size=10
Auth: type=status-dot, style=sonar, color=#f59e0b, position=left, rippleCount=2, label="Auth Service"
Payment: type=status-dot, style=blink, color=#ef4444, position=left, label="Payment — down"
legend dot: position=below-center, padding=16
```

**Animation styles:**

| Style | Effect | Use case |
|-------|--------|----------|
| `breathe` | Smooth opacity oscillation | Soft, persistent indicator |
| `blink` | Hard on/off strobe | Urgent alert, error |
| `sonar` | Expanding stroked rings | Broadcasting, active transmission |
| `ripple` | Expanding filled semi-transparent rings | Softer broadcast |

**`legend dot:` line**

When present, renders a floating chip row showing nodes that have a `label=` value. Nodes without `label=` are omitted. The legend is not included in SVG/PNG exports.

```text
legend dot: position=below-center, padding=16
```
